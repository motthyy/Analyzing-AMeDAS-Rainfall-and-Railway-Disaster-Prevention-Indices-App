# 計算仕様書

本書は、本アプリが実装する全ての数式・初期値・係数・更新順序・欠測処理を記載する。

## 1. 前処理

### 1.1 24:00表記の正規化

気象庁CSV（`ymdLiteral=0`指定）では、時刻は年・月・日・時（1〜24）の4列に分割されて得られる。時=24は当日24:00＝翌日00:00として扱う。

```
if hour == 24:
    datetime = date + 1日, 00:00
else:
    datetime = date, hour:00
```

内部日時はすべてタイムゾーン付きの日本標準時（`Asia/Tokyo`）として保持する。

### 1.2 閾値処理（0.3mm）

原時雨量 `R_t` から計算用時雨量 `r_t` を次式で求める。

```
r_t = 0        (R_t <= 0.3)
r_t = R_t      (R_t > 0.3)
```

欠測（NaN）はこの変換の対象外とし、NaNのまま保持する（0へは変換しない）。

## 2. 重複時刻の統合（複数ダウンロードの統合）

同一時刻に複数の観測値候補がある場合、以下の優先順位で1件を採用する。

1. 品質情報が「正常(8)」の値
2. 品質情報が「準正常(5/4/2)」の値
3. 品質情報が「欠測(1)」または「非対象(0)」の値

同一優先順位内で値が異なる場合は「競合」として記録し、いずれか（先着）を代表値として採用しつつ、`is_conflicting=True` を立てて黙って上書きしない。

## 3. 12時間無降雨リセット連続雨量

状態変数: 累積雨量 `C`、無降雨継続時間 `D`（時間単位の整数）、イベント番号 `E`。

初期値: `C=0`, `D=dry_hours_reset(=12)`（開始時点は「十分乾燥している」とみなす）, `E=0`。

各時刻 `t` について、計算用時雨量 `r_t` が有効値の場合:

```
if r_t > 0:
    if D >= 12:                # 12時間以上の無降雨明けの降雨 → 新イベント
        E = E + 1
    C = C + r_t
    D = 0
else:
    D = D + 1
    if D >= 12:
        C = 0                  # 12時間目の無降雨でリセット
```

`r_t` が欠測（NaN）の場合、その時刻の出力はすべてNaNとし、次の有効値で `C=0, D=12` に再初期化した上で `state_reset_due_to_gap=True`, `warmup_flag=True` を立てる。

## 4. 24時間移動雨量

```
R24(t) = Σ_{i=0}^{23} r_{t-i}
```

直近24時間分すべてが有効値でない場合（欠測を含む場合、またはデータ先頭で24時間分に満たない場合）はNaNとする。

## 5. 実効雨量

半減期 `H`（1.5時間・6時間・24時間）に対する1時間あたりの残存率:

```
a_H = 0.5^(1/H)
```

漸化式:

```
E_H(t) = r_t + a_H * E_H(t-1)
```

初期値: `E_H(開始時)=0`。欠測明けの最初の有効時刻でも `E_H=0` から再初期化し、`state_reset_due_to_gap`, `warmup_flag` を立てる。

## 6. 推定10分雨量

計算用時雨量 `r_t` を6等分し、その1時間に均等に降ったと仮定する。

```
r_t = 0     の場合: R10_1..6 = 0,0,0,0,0,0
r_t > 0     の場合: R10_i = r_t / 6  (i=1..6)
```

時刻 `t` の値は、区間 `(t-50分, t-40分, ..., t)` の6個の10分値に対応する（気象庁の時別値は「その時刻までの1時間」の積算値であるため）。

## 7. 気象庁標準3段タンクモデル（推定土壌雨量指数）

**出典・注意**: 気象庁が技術資料等で示す標準3段タンクモデルの構造（3段直列タンク、各段の側方流出孔・側方流出係数・底面浸透係数）を参考に、`config/tank_model.yaml` へ係数を設定している。ただし本アプリの入力は気象庁公表の10分雨量ではなく、上記6節の「1時間雨量の均等6分配」で推定した値であるため、**算出される指数は気象庁が公式に公表する土壌雨量指数と同一ではない**。本アプリでは常に「推定土壌雨量指数」と呼称する。

### 7.1 係数（`config/tank_model.yaml`）

| タンク | 側方流出孔高さ(mm) | 側方流出係数(1/h) | 底面浸透係数(1/h) |
|---|---|---|---|
| 第1タンク | 15 / 60 | 0.10 / 0.15 | 0.12 |
| 第2タンク | 15 | 0.05 | 0.05 |
| 第3タンク | 15 | 0.01 | 0.01 |

時間刻み: `Δt = 1/6時間`（10分）。初期貯留量: 全タンク0mm。

### 7.2 更新順序（10分ステップごと）

各タンクについて、(1)入力を加算 → (2)側方流出量・底面浸透量を現在の貯留高から計算 → (3)貯留量から両方を差し引く（0未満にならないようクリップ）、の順で計算する。

```
タンク1:
    S1 = S1_prev + 入力(降雨量)
    側方流出1 = Σ_{孔 h, k} [ S1>h ? k*(S1-h)*Δt : 0 ]
    浸透1     = 浸透係数1 * S1 * Δt
    S1_new = max(S1 - 側方流出1 - 浸透1, 0)

タンク2:
    S2 = S2_prev + 浸透1        # タンク1の浸透がタンク2への入力
    側方流出2, 浸透2 を同様に計算
    S2_new = max(S2 - 側方流出2 - 浸透2, 0)

タンク3:
    S3 = S3_prev + 浸透2        # タンク2の浸透がタンク3への入力
    側方流出3, 浸透3 を同様に計算（浸透3は系外への損失＝深部浸透とみなす）
    S3_new = max(S3 - 側方流出3 - 浸透3, 0)
```

10分刻みで6回更新し、各時間の6回目（＝当該時刻ちょうど）の更新後状態を時別データとして採用する。

推定土壌雨量指数:

```
S(t) = S1(t) + S2(t) + S3(t)
```

欠測明けの最初の有効時刻では、3タンクとも0から再初期化する。

## 8. 年最大値・年区切り

### 8.1 年区切り定義

| キー | 名称 | 開始 |
|---|---|---|
| calendar | 暦年 | 1月1日 |
| fiscal | 年度 | 4月1日 |
| june_start | 6月始まり年 | 6月1日 |

年区分名は開始年で表す（例: 2025年度＝2025-04-01〜2026-03-31）。各時刻がどの年区分に属するかは、`(月,日) >= (開始月,開始日)` なら当該年、そうでなければ前年、として判定する。

### 8.2 データ完全性

```
完全率(%) = 100 * 有効時間数 / 想定時間数
```

既定の採用可否閾値は95%。以下は既定で除外する（ダッシュボード上で変更可能）。

- 観測開始を含む不完全年
- 実行時点で終了していない最新年区分
- 完全率が閾値未満の年
- 状態量（連続雨量・実効雨量・タンクモデル等）が大きな欠測により再初期化された区間を含む年

## 9. ガンベル分布

分布関数: `F(x) = exp[-exp(-(x-μ)/β)]`

### 9.1 最尤法

SciPy `scipy.stats.gumbel_r.fit(data)` を用いる。

### 9.2 積率法

Excel（`r_max_c(manual ver.).xlsm`のrp_inシート）の計算式と完全一致させている。

```
β = sqrt(6) * s / π       (sは母標準偏差。ExcelのSTDEV.P相当、ddof=0)
μ = mean - 0.5772 * β      (0.5772はExcelが用いる定数。オイラー・マスケローニ定数の
                            厳密値0.5772156649ではなく4桁の近似値をそのまま使う)
```

### 9.3 確率雨量

```
x_T = μ - β * ln[-ln(1 - 1/T)]     (T > 1年)
x_1 = 算出不可（NaN、T=1のとき ln(0) が定義できないため）
```

出力する確率年: 1〜30年（1年刻み）、50・100・200・500年。

### 9.4 プロッティングポジション

```
Gringorten: F_m = (m - 0.44) / (n + 0.12)
Weibull:    F_m = m / (n + 1)
Cunnane:    F_m = (m - 0.4) / (n + 0.2)
T_m = 1 / (1 - F_m)
```
（mは昇順順位 1..n）

### 9.5 ブートストラップ信頼区間

年最大値をリサンプリング（復元抽出、標本サイズn）し、都度ガンベル分布を再推定して `x_T` を計算する処理を既定1000回繰り返し、指定信頼水準のパーセンタイルを信頼区間とする。乱数シードは指定可能（既定42）で、同一シードでは結果が再現される。

リサンプルが退化した標本（分散ゼロに近い等）になり、SciPyの最適化が数値的に破綻する場合（`OverflowError`/`RuntimeError`等）は、当該反復をスキップする。

### 9.6 適合度

- AIC = `2*2 - 2*対数尤度`
- Kolmogorov-Smirnov統計量（`scipy.stats.kstest`）
- RMSE・相関係数: プロッティングポジションによる経験的再現期間に対する適合値との比較

## 10. 標本数・外挿に関する警告

- 観測年数が10年未満: 「解析に注意が必要」
- 観測年数が10年以上20年未満: 「不確実性大」
- 確率年が観測年数の3倍を超える: 「大幅な外挿」
