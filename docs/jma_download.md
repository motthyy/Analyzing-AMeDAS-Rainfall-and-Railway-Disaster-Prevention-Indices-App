# 気象庁データ取得の実装仕様（現行サイト調査結果）

**調査日: 2026年7月18日。** 気象庁サイトの仕様は予告なく変更される可能性があるため、
本書の内容は将来にわたって保証されるものではない。取得に失敗するようになった場合は、
本書とともに以下のモジュールを見直すこと。

- `src/amedas_rainfall/jma/direct_client.py`
- `src/amedas_rainfall/jma/playwright_client.py`
- `src/amedas_rainfall/jma/csv_parser.py`
- `src/amedas_rainfall/jma/station_catalog.py`

調査は、実際にPowerShell/Pythonから本番サーバーへHTTPリクエストを送信し、
レスポンスを確認する形で行った（推測やサードパーティ記事の孫引きではない）。

## 1. ページ・アセットの構成

- HTMLページ: `https://www.data.jma.go.jp/gmd/risk/obsdl/index.php`
- クライアントロジック本体（jQuery）: `https://www.data.jma.go.jp/risk/obsdl/web/js/top.2.1.js`
- **重要**: HTMLページのパスは `gmd/risk/obsdl/` だが、AJAX/CSVエンドポイントは
  `gmd` を含まない `risk/obsdl/` 配下にある（JS内 `rooturl = "//" + location.hostname + "/risk/obsdl/"`）。

## 2. セッション確立

`GET https://www.data.jma.go.jp/gmd/risk/obsdl/index.php` へアクセスすると、
CodeIgniterベースの `ci_session` Cookie（有効期限約10分）とAWS ALBの
スティッキーセッションCookie（`AWSALB`, `AWSALBCORS`）が発行される。
以降のPOSTリクエストはこのCookieを保持したセッションで行う必要がある
（`JmaDirectClient.ensure_session()` が対応）。

## 3. 地点マスタ取得

標準的なJSON/CSV/KML形式の地点マスタファイルは存在しない。地点一覧は
都道府県ごとのHTMLフラグメントをPOSTで取得し、DOMを解析して得る。

```
POST https://www.data.jma.go.jp/risk/obsdl/top/station
  body: pd=00                  → 都道府県コード(prid)一覧のHTML
POST https://www.data.jma.go.jp/risk/obsdl/top/station
  body: pd=<prid>               → 当該都道府県の地点一覧HTML
```

レスポンスの文字コードは **UTF-8**（CSVダウンロードとは異なる）。

地点一覧HTMLは、地点ごとに2つの`<div class="station ...">`（当たり判定用と
地図マーカー用）を含み、以下の情報を持つ。

- `title`属性: 複数行テキストで地点名・カナ・緯度（度分）・経度（度分）・標高、
  廃止地点の場合は「YYYY年M月D日に観測終了」の行が追加される。
- 隠しinput: `stid`（地点ID）, `stname`（地点名）, `prid`（都道府県コード）,
  `kansoku`（6桁の観測フラグ文字列）

`stid` の命名規則: `s` + 5桁 = 気象官署（例: `s47662`=東京）、
`a` + 4桁 = アメダスのみの地点（例: `a0365`=小河内）。

`kansoku` の各桁（`1`または`2`で観測あり、`0`で観測なし）:
`[降水量, 風, 気温, 日照, 積雪, その他]`。降水量観測の有無は1桁目で判定する。

廃止地点は、当たり判定divのclassに `owata` が含まれる（例:
`class="station owata"`）ことに加え、title属性内の観測終了日行で判定できる。

## 4. 時別降水量CSVのダウンロード

```
POST https://www.data.jma.go.jp/risk/obsdl/show/table
```

主要パラメータ（本アプリの`direct_client.py`が送信する値）:

| パラメータ | 値の例 | 備考 |
|---|---|---|
| stationNumList | `["a0365"]` | JSON文字列化した配列 |
| aggrgPeriod | `"9"` | 9=時別値 |
| elementNumList | `[["101",""]]` | 101=降水量（前1時間）。第2要素は閾値等がない要素では空文字 |
| interAnnualType | `"1"` | 1=連続期間 |
| ymdList | `["開始年","終了年","開始月","終了月","開始日","終了日"]` | **すべて文字列**。数値のまま送るとサーバーに拒否される |
| optionNumList | `[]` | 平年値比較等（未使用） |
| downloadFlag | `"true"` | trueでCSV実体、falseでプレビュー用JSON |
| rmkFlag | `"1"` | 品質情報の注記を含める |
| disconnectFlag | `"1"` | 観測環境変化前後のデータを含める（均質番号を表示） |
| csvFlag | `"1"` | 品質情報・均質番号を独立した列として出力（数値のみ） |
| ymdLiteral | `"0"` | **本アプリは0を指定**。年・月・日・時を分割した列で取得し、 24時表記を自前で正規化する（1指定時は日付列が結合され24時が自動的に翌日0時へ変換される） |
| youbiFlag / fukenFlag / kijiFlag | `"0"` | 曜日・都道府県名・現象時刻列は使用しない |
| jikantaiFlag / jikantaiList | `"0"` / `[]` | 時間帯の絞り込みは使用しない |

レスポンス: `Content-Type: application/octet-stream`、
`Content-Disposition: attachment; filename="data.csv"...`、本文がそのままCSVバイト列
（**Shift_JIS(CP932)エンコード**）。ポーリング不要で、POSTの応答に同期的に含まれる。

CSRFトークンは存在しない。ヘッダーは `Content-Type: application/x-www-form-urlencoded`
のみで機能することを確認済み（`Referer`ヘッダーは念のため付与している）。

## 5. CSVフォーマット（`ymdLiteral=0`時）

```
ダウンロードした時刻：2026/07/18 12:17:17
                                          ← 空行
,,,,豊田,豊田,豊田                        ← 地点名（日付列分の空セル＋要素列ごと）
年,月,日,時,降水量(mm),降水量(mm),降水量(mm)  ← 日付見出し＋要素名
,,,,,品質情報,均質番号                     ← 副見出し（値列は空）
2024,1,1,1,0.0,8,1                        ← データ行
...
2024,1,1,24,0.0,8,1                       ← 24時＝当日24:00（自前で翌日0:00へ正規化）
```

要素グループの列数は、気象官署など「現象なし情報」を持つ要素種別では4列
（値・現象なし情報・品質情報・均質番号）、アメダスの降水量など持たない場合は3列
（値・品質情報・均質番号）になる。`csv_parser.py`は副見出し行のラベル文字列
（「品質情報」「均質番号」「現象なし情報」）を検出して列位置を決定するため、
どちらの形式でも解析できる。

### 品質情報コード

| コード | 意味 |
|---|---|
| 8 | 正常 |
| 5 | 準正常（必要資料の80%以上が存在） |
| 4 | 準正常（資料不足） |
| 2 | 値が疑わしい（時別値のみ） |
| 1 | 欠測 |
| 0 | 非対象（その時刻には観測自体が行われていない＝観測開始前や地点種別上対象外） |

品質コード「0」（非対象）と「1」（欠測）はいずれも値が空欄になるが、
本アプリの開始日時探索（`start_date_finder.py`）では、「0以外」を
「観測が行われていた証拠」として区別に利用している。

### 均質番号

ダウンロードの都度、期間内で観測環境変化が検出されるたびに1から連番が振られる。
**異なるダウンロード間で値を比較・突合する用途には使えない**。

## 6. 負荷対策・利用規約

- `robots.txt` は404（機械可読のクロール制限は存在しない）。
- ただし地点一覧HTML内に「自動化ツール等による過度のアクセスはお控えいただくよう
  お願いいたします」という明示的な注意書きがある。
- 同日データは取得不可（前日まで）。
- クライアント側に `seigen=44000` という行数上限を示唆する変数があり、
  長期間・複数地点・複数要素を一括指定すると制限にかかる可能性がある
  （本アプリは1地点・1要素・原則1年単位で分割するため、通常は問題にならない）。

これらを踏まえ、本アプリは既定で3秒（最小2秒）の待機時間・直列リクエスト・
指数バックオフ・既取得済み期間の再取得スキップを実装している
（`config/default.yaml` の `download` セクション）。

## 7. 未検証・要注意事項

- 官署種別（`s`プレフィックス）の地点における「現象なし情報」列の実データでの
  挙動は、ヘルプページの説明に基づく実装であり、実際の官署CSVでの動作確認は
  限定的である。
- 全国の全都道府県分の地点一覧取得は、東京都・一部近隣県のみ実地点で確認済みで、
  全件の網羅的な確認は行っていない。
