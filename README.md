# アメダス長期雨量・鉄道防災指標解析アプリ

気象庁「[過去の気象データ・ダウンロード](https://www.data.jma.go.jp/gmd/risk/obsdl/index.php)」から、任意のアメダス／気象官署の時別降水量を取得し、鉄道防災用の雨量指標・推定土壌雨量指数・ガンベル分布による確率雨量を計算し、Streamlit上で可視化・画像出力できるアプリケーションです。

**表示言語は日本語のみです。** インターネット接続が必要で、気象庁以外の非公式データサイトは使用していません。

このマニュアル（README.md）は、アプリ起動後の「マニュアル」タブからも直接閲覧できます。

---

## 1. アプリ概要

1. 地点マスタ（都道府県・地点一覧）を取得・キャッシュ
2. 選択した地点の時別降水量が取得可能な最古の日時を探索
3. 1年単位を基本に、気象庁サイトへの負荷に配慮しながら全期間をダウンロード
4. 複数回のダウンロード結果を、品質情報に基づき1つの連続時系列へ統合
5. 鉄道防災用の各種雨量指標・推定土壌雨量指数を計算
6. 暦年／年度／6月始まり年の3種類で年最大値を集計
7. ガンベル分布により確率雨量（再現期間ごとの見込み雨量）を推定
8. Plotlyによるインタラクティブなグラフ表示と、PNG/SVG/PDFでの画像出力

## 2. Python 3.11の確認方法

PowerShellで以下を実行してください。

```powershell
py -3.11 --version
```

`Python 3.11.x` と表示されればOKです。表示されない場合は [python.org](https://www.python.org/downloads/) からPython 3.11をインストールしてください（インストール時に「Add python.exe to PATH」を有効にすることを推奨します）。

## 3. インストール

プロジェクトフォルダで `install.bat` をダブルクリックするか、PowerShellで実行してください。

```powershell
.\install.bat
```

実行すると、最初に次の2択を聞かれます（`1`または`2`を押すだけで選べます。Enterキーは不要です）。

- **[1] Python 3.11を自動的にインストールしてから、必要ライブラリを導入する**
  Python 3.11が入っていないPCで、環境構築ごと自動で行いたい場合に選びます。
  公式サイト(python.org)からインストーラーを自動ダウンロードし、現在のユーザー用として
  （管理者権限不要で）インストールします。
- **[2] Python 3.11はインストール済み — 必要ライブラリのみ導入する**
  既にPython 3.11が入っている場合はこちらを選びます（2節の確認方法で確認できます）。

どちらを選んでも、その後は自動的に以下を行います。

- ローカル仮想環境 `.venv` を作成します。
- `requirements.txt` に基づき依存ライブラリを導入します。
- Playwright（予備ダウンロード手段）のChromiumブラウザ本体も導入を試みます（失敗しても直接ダウンロード方式には影響しません）。

他の人にこのアプリを配布する場合、相手のPCにPython 3.11が入っているか分からなくても、
`install.bat`を渡して選択肢[1]を選んでもらうだけで環境構築が完了します。

## 4. 起動

```powershell

```

または、仮想環境を有効化した上で:

```powershell

```

## 5. 地点選択

「地点選択・ダウンロード」タブで:

1. 「地点マスタを更新」を押すと、気象庁サイトから全都道府県の地点情報を取得します（初回のみ必要。数分かかります）。以降は `data/station_master/stations.parquet` のキャッシュを使用します。
2. 都道府県・地点名（部分一致）・地点コード・現在観測中のみ／降水量観測地点のみ・地点種別で絞り込みます。
3. 一覧から地点を1つ選択すると、緯度経度がPlotly地図（OpenStreetMap、無料・APIキー不要）に表示されます。

## 6. ダウンロード

1. 「開始年を探索する」を押すと、地点種別に応じた候補年から、時別降水量が実際に取得できる最古の年・日時を探索します（4節の推奨手順に基づき、10年→5年→1年単位で絞り込み、非単調な欠測パターンにも配慮します）。開始年は手動でも修正できます。探索が完了すると、ダウンロード計画の開始日は既定でその**探索結果の年の1月1日**に設定されます（探索で見つかった正確な日時ではなく、年単位で切りよくダウンロードするため）。必要に応じて手動で変更できます。
2. 「ダウンロード計画を作成/更新」を押すと、1年単位のジョブ（SQLite `state/jobs.sqlite` で管理）を作成します。
3. 赤色で強調された「ダウンロード開始」ボタンを押すと、1件ずつ取得・保存の成功を確認しながら、最新データまで自動的に連続して取得します。進捗バーと「完了件数/合計件数（％）」がリアルタイムに表示され、完了時には完了メッセージが表示されます。気象庁サイトへの負荷対策として、リクエスト間に待機時間（既定3秒）を挟み、失敗時は6か月→3か月→1か月→7日の順に自動分割し、それでも失敗した期間は再試行待機（既定10秒→30秒→120秒、指数バックオフ）を経てFAILEDとして記録した上で、次の期間へ自動的に進みます。
4. ダウンロードしたCSVは `data/raw/{地点コード}_{地点名}/` にそのまま保存されます。
5. 「データ解析」を押すと、生CSVを解析・統合して `data/normalized/{地点コード}/hourly.parquet` を作成し、続けて全指標（連続雨量・実効雨量・推定土壌雨量指数など）を計算して `data/calculated/{地点コード}/indices.parquet` に保存します。計算には時間がかかるため（推定土壌雨量指数の計算が特に重く、10分刻みで数十年分を処理します）、進捗バーと進捗率（％）が表示されます。一度保存すれば、次回以降は他のタブを開いてもキャッシュから瞬時に読み込まれ、再計算は行われません（新しいデータをダウンロードして正規化データが更新された場合のみ自動的に再計算されます）。

## 7. 中断・再開

ジョブの状態はすべてSQLite（`state/jobs.sqlite`）に保存されます。アプリを閉じても、次回起動時に「ダウンロード開始」を押せば、未完了（PENDING/RETRY_WAIT）のジョブから自動的に再開されます。「失敗期間を再試行対象に戻す」ボタンで、FAILEDになった期間を再試行対象に戻せます。

## 8. 指標の意味

| 指標 | 説明 |
|---|---|
| 時雨量 (`rainfall_raw_mm`) | 気象庁から取得した時別降水量そのもの（グラフ上では単に「時雨量」と表記） |
| 閾値処理後時雨量 (`rainfall_used_mm`) | 後述の0.3mm閾値処理を適用した値 |
| 12時間無降雨リセット連続雨量 | 一連の降雨イベントの累積雨量。無降雨が12時間続くと0にリセット |
| 24時間移動雨量 | 各時刻を終端とする直近24時間の合計（24時間分揃わない場合はNaN） |
| 実効雨量（半減期1.5h/6h/24h） | 指数関数的に古い雨を減衰させながら積算する雨量指標 |
| 推定土壌雨量指数 | 気象庁標準3段タンクモデルに基づく独自推定値（後述） |

## 9. 0.3mm閾値

気象庁の時別降水量が0.3mm/h以下の場合、無降雨とみなして計算用時雨量を0とします（0.4mm/h以上はそのまま採用）。欠測値はこの閾値処理の対象外で、0へは変換せずNaNのまま保持します。

## 10. 10分雨量の均等分配仮定

気象庁からは時別値のみを取得します。推定土壌雨量指数の計算には10分刻みの入力が必要なため、閾値処理後の1時間雨量を単純に6等分し、その1時間の間に均等に降ったものと仮定して10分雨量を推定しています。実際の降雨は1時間内で一様ではないため、この仮定による誤差が生じます。

## 11. 推定土壌雨量指数は気象庁公表値ではありません

本アプリの「推定土壌雨量指数」は、気象庁が公式に算出・公表している土壌雨量指数**そのものではありません**。上記の10分雨量均等分配という仮定と、`config/tank_model.yaml` に設定した3段タンクモデル係数（気象庁標準タンクモデルの構造を参考に設定）を用いて、本アプリが独自に計算した推定値です。画面・出力ファイル上でも「気象庁公表土壌雨量指数」とは表記していません。詳細な計算式は `docs/calculation_method.md` を参照してください。

## 12. ガンベル法

年最大値系列から、最尤法（SciPy `gumbel_r`）または積率法（数式による独自実装）でガンベル分布のパラメータ（位置パラメータμ、尺度パラメータβ）を推定し、任意の確率年Tに対する確率雨量を

```
x_T = μ - β・ln[-ln(1 - 1/T)]
```

で計算します。プロッティングポジションはGringorten法を標準とし、Weibull法・Cunnane法も選択できます。確率年は1〜30年・50年・100年までを算出対象とします（信頼区間表示は行いません）。

## 13. 確率年1年を算出できない理由

ガンベル分布の再現期間の式では、T=1年のとき `1 - 1/T = 0` となり、`ln(0)` が定義できず有限の値が求まりません。そのため、確率年1年は表には表示しますが値は「算出不可」とし、数値を捏造しません。グラフの横軸も1年より大きい範囲（既定1.05年〜）から描画します。

## 14. 欠測処理

- 欠測は0mmとして扱いません（原雨量・計算用雨量ともにNaNのまま保持）。
- 複数ファイルに同一時刻の値が存在する場合、正常品質値→準正常値→欠測値の優先順位で統合し、品質が同一で値が異なる場合は「競合」として記録します（黙って上書きしません）。
- 連続雨量・実効雨量・タンクモデルなどの状態量は、欠測明けの最初の有効時刻で0から再初期化し、`state_reset_due_to_gap` / `warmup_flag` を立てます。
- 年最大値解析では、年間データ完全率（既定95%）未満の年、観測開始を含む不完全年、実行時点で終了していない最新年区分は既定で除外対象とし、ダッシュボード上で採否を変更できます。

## 15. 画像出力

「時系列グラフ」「確率雨量グラフ」タブのグラフ調整パネルで図幅・DPI・フォント・線種・色（白黒モード含む）等を調整し、「画像を生成してダウンロード用に保存」でPNG（300/600/1200dpiまたは任意）・SVG・PDFを `output/figures/` に出力できます（Plotly＋Kaleidoによるサーバーサイド再生成）。グラフ設定は `output/plot_settings/` にJSONとして保存・再読込できます。

グラフはブラウザのダーク／ライトテーマに関わらず白背景・黒文字で統一表示され、枠線付きで読みやすくしています。グラフ上にカーソルを合わせると、縦横のクロスヘア線とその時刻・数値が表示されます（画像出力時は、代わりに目盛線・軸範囲を明示することで値を読み取れるようにしています）。

### 時系列グラフの表示期間

「時系列グラフ」タブの表示期間は、プリセット（最新31日／今月／前月／任意期間）から選べます。プリセットを選んでいる間は開始・終了日時欄は編集不可となり、選んだプリセットに応じた日時が自動的に表示されます。任意の期間を見たい場合は「任意期間」を選んで開始・終了日時を直接指定してください。

上段（棒グラフ）は既定で「時雨量」、下段（折れ線グラフ）は既定で「推定土壌雨量指数」が選択されています。いずれも複数指標や別の指標に変更できます。

### 大量データ表示時の間引き（ピーク保持）

数十年分の時別データをそのまま表示すると、ブラウザでの描画が重くなります。表示対象が約2万点を超える場合、区間ごとの**最大値を保持したまま**間引いて表示します（単純な間引きではなく最大値を採用することで、大雨イベントのピークがグラフから消えないようにしています）。間引きが行われた場合は、グラフ右上に「表示高速化のため◯◯点を◯◯点に間引いて表示中」という注記が表示されます。詳細な時刻レベルの確認が必要な場合は、表示期間を絞り込んでください（画像出力も、その時点で画面に表示されている間引き後のデータに基づきます）。

### データ出力（CSV/Parquet/Excel）の進捗表示

「データ出力」タブの各出力ボタン（時別データ・年最大値・全項目まとめのExcelブック）を押すと、「ダウンロード中...」という状態表示とともに進捗バー・進捗率（％）がリアルタイムに表示されます。数十年分のデータではExcel書き出しに時間がかかることがあるため、画面が固まったように見えないよう、書き出し中のファイル種別（Parquet/CSV/Excel、Excelは複数シート単位）ごとに進捗を更新します。完了すると進捗表示は消え、出力先パスを含む完了メッセージが表示されます。

## 16. データ保存場所

```
data/station_master/stations.parquet   地点マスタキャッシュ
data/raw/{地点コード}_{地点名}/         生CSV（未加工）
data/normalized/{地点コード}/hourly.parquet  正規化済み時別データ
data/calculated/{地点コード}/indices.parquet  計算済み全指標（連続雨量・実効雨量・推定土壌雨量指数等）のキャッシュ
state/jobs.sqlite                       ダウンロードジョブ管理
output/csv/ output/excel/ output/figures/ output/plot_settings/  各種出力
logs/app_YYYYMMDD.log                   ログ
```

## 17. トラブルシューティング

| 症状 | 対処 |
|---|---|
| `py -3.11` が見つからない | Python 3.11を公式サイトからインストールし、PATHを通してください |
| install.batで依存関係のインストールに失敗する | ネットワーク接続を確認し、再実行してください。社内プロキシ環境ではpipのプロキシ設定が必要な場合があります |
| ダウンロードが進まない/失敗が多い | 待機時間を長くする、時間を置いて再試行してください。気象庁サイト側の一時的な混雑の可能性があります |
| Playwright予備手段が使えない | `.venv\Scripts\python.exe -m playwright install chromium` を再実行してください |
| 日本語フォントが文字化けする | Windows標準のYu Gothic/Meiryoが利用できない環境では自動的にsans-serifへフォールバックします |
| Excel出力が省略される | 時別データの行数がExcelの上限(約104万行)に近い場合、Excelシートは省略されCSV/Parquetの利用が案内されます |
| アプリを更新したのに画面が変わらない／`removeChild`等のエラーが出る | `run.bat`を複数回実行すると古いプロセスが残ったままになり、ブラウザが古い方につながり続けることがあります。タスクマネージャーで残っている`python.exe`（コマンドラインに`streamlit`を含むもの）を全て終了し、`run.bat`を1つだけ起動し直した上で、ブラウザを再読み込み（Ctrl+F5）してください |

## 18. 気象庁サイトの仕様変更に備えて

気象庁のサイト構造やAPIパラメータが変更された場合、修正が必要となる可能性が高いモジュールは以下です。

- `src/amedas_rainfall/jma/direct_client.py`（HTTPリクエスト方式・パラメータ）
- `src/amedas_rainfall/jma/playwright_client.py`（ブラウザ操作予備手段）
- `src/amedas_rainfall/jma/csv_parser.py`（CSVフォーマット）
- `src/amedas_rainfall/jma/station_catalog.py`（地点一覧HTMLの構造）

現行仕様の詳細な調査結果は `docs/jma_download.md` を参照してください。

## 19. 利用上の注意

- 本アプリは気象庁の利用上の注意（自動化ツールによる過度なアクセスの抑制を求める旨の記載）を尊重し、既定で3秒（最小2秒）の待機時間・直列リクエスト・指数バックオフを実装しています。待機時間を極端に短く設定しないでください。
- 気象庁の観測データはPublic Data License等の下で公開されていますが、二次利用時は気象庁公式サイトの利用規約を必ずご確認ください。
- 本アプリはAPIキーや有料サービスを必要とせず、認証情報もコードへ埋め込んでいません。
- 「推定土壌雨量指数」は気象庁公表値ではない独自推定値です（11節参照）。

## 20. 未検証事項（重要）

このプロジェクトはPython/PowerShell環境で実データに対する動作確認を行っています（`docs/validation.md` に詳細を記載）。ただし以下は未検証、または限定的な検証にとどまっています。

- 全国の全都道府県・全地点に対する地点マスタ一括取得（1〜2都道府県分のみ実地点で確認）。
- 開始年探索アルゴリズムの、実際に長期欠測がある地点での網羅的な検証。
- Playwright予備手段の実際の動作（direct_clientの実データ取得確認は行いましたが、playwright_clientは同一ロジックのコードレビューのみで、実行時検証はしていません）。
- 数十年規模の長期データでのExcel出力・全項目Excelブックの実行時間。
- Windows以外のOSでの動作（本アプリはWindows 10/11を前提としています）。

これらの箇所を「動作確認済み」とは表現していません。ご利用の際は、まず1地点・数年分の小規模なデータで一連の操作（地点選択→ダウンロード→正規化→指標計算→年最大値→確率雨量）を確認することを推奨します。

## 21. 最初に試す推奨手順

1. `install.bat` を実行
2. `run.bat` でアプリを起動
3. 「地点選択・ダウンロード」タブで「地点マスタを更新」（初回のみ）
4. 都道府県「東京都」、地点名に「豊田」または地点コード `a0365`（小河内）で検索し選択
5. 「開始年を探索する」→「ダウンロード計画を作成/更新」→「次のバッチを実行」を数回押して数年分を取得
6. 「正規化データを再構築」
7. 「データ品質」タブで欠測・完全率を確認
8. 「時系列グラフ」タブでグラフを確認・画像出力
9. 「確率雨量グラフ」タブで採用年を確認しつつガンベル分布による確率雨量を確認
